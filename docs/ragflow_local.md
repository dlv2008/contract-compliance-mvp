# 本地 RAGFlow 接入说明

## 目标

把 `MVP` 接到你本地 `WSL` 中已经存在的 `RAGFlow`，不重新设计一套新的知识库底座。

## 已发现的本地目录

- 当前源码目录：`~/ragflow`
- 旧版本目录：`~/ragflow_0.19`

## 本项目对 RAGFlow 的最低要求

1. 能创建知识库
2. 能导入 `resource/02_制度样本` 和 `resource/01_合同样本`
3. 能按问题返回相关制度片段
4. 能给出文档片段或条款级引用

## 建议先做的 4 个知识库

1. `财务付款制度库`
2. `采购制度库`
3. `合同模板库`
4. `案例与报告样例库`

## 建议保留的连接信息

在本项目中统一记录：

- `RAGFLOW_BASE_URL`
- `RAGFLOW_API_KEY`
- 各知识库的 `dataset_id`
- 主要解析配置

## 最小验证口径

### 验证 1：服务进程

```bash
ps -ef | grep -E 'ragflow_server.py|task_executor.py' | grep -v grep
```

### 验证 2：后端监听

```bash
ss -ltnp | grep 9380
```

### 验证 3：知识库效果

至少用下面这类问题测通：

- `预付款比例上限是多少`
- `合同里没有发票条款应如何提示`
- `收款账户与签约主体不一致有什么风险`
- `自动续约条款需要如何提示`

## 当前结论

- `RAGFlow` 继续做知识库、解析、引用底座
- 不把 `RAGFlow` 当作整个审查系统
- 审查系统自己的结果页、复核页、报告页仍由本项目负责
