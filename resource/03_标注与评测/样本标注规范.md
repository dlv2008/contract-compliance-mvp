# 样本标注规范

本规范用于给合同样本做 gold label，供：

- 字段抽取评测
- 风险命中评测
- 证据定位评测

## 1. 标注单位

标注分 3 层：

1. 合同级
2. 字段级
3. 风险级

## 2. 合同级标注

必须标：

- `sample_id`
- `contract_type`
- `expected_overall_risk`
- `expected_decision`

示例：

```json
{
  "sample_id": "svc_b_001",
  "contract_type": "service_contract",
  "expected_overall_risk": "red",
  "expected_decision": "manual_review"
}
```

## 3. 字段级标注

每个字段必须包含：

- `field`
- `value`
- `status`
- `evidence_clause_ids`

其中：

- `status=present`：字段存在且明确
- `status=missing`：字段应有但缺失
- `status=conflict`：字段存在多个冲突值
- `status=uncertain`：字段表述模糊

示例：

```json
{
  "field": "payment.prepay_ratio",
  "value": 50,
  "status": "present",
  "evidence_clause_ids": ["C014"]
}
```

## 4. 风险级标注

每条风险必须包含：

- `rule_id`
- `risk_level`
- `hit`
- `message`
- `contract_evidence_clause_ids`
- `policy_reference_ids`

示例：

```json
{
  "rule_id": "FIN-SVC-004",
  "risk_level": "red",
  "hit": true,
  "message": "预付款比例超过30%，且合同以验收为付款前提",
  "contract_evidence_clause_ids": ["C014", "C015"],
  "policy_reference_ids": ["POLICY-FUND-003"]
}
```

## 5. clause_id 标注要求

合同样本中的条款统一使用 `C001`、`C002` 这类编号。

标注时只引用条款编号，不复制整段原文。

## 6. 推荐文件格式

每个样本一份 `.annotation.json`

顶层结构建议如下：

```json
{
  "sample_id": "svc_b_001",
  "meta": {
    "contract_type": "service_contract",
    "expected_overall_risk": "red",
    "expected_decision": "manual_review"
  },
  "fields": [],
  "risks": []
}
```

## 7. 标注完成标准

一份样本只有同时满足下面 4 条才算完成：

1. 关键字段全部标完
2. 缺失字段已显式标记
3. 风险项已绑定规则编号
4. 每条风险都已绑定证据条款
