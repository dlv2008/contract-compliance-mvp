# 财务付款管理制度（MVP版）

【POLICY-FUND-001】付款前置条件  
任何合同项下付款均应以合同生效、预算占用、审批完成和付款资料齐备为前提。

【POLICY-FUND-002】发票要求  
合同约定应取得发票的，付款前必须取得合法有效发票；发票类型、税率和开票内容应与合同及业务实质一致。

【POLICY-FUND-003】预付款控制  
服务类合同预付款比例原则上不得超过合同总价的 30%；确需超过的，应履行例外审批并由业务部门说明必要性。

【POLICY-FUND-004】验收与付款衔接  
以交付成果、服务完成或设备到货作为付款依据的，应先完成验收，再支付对应尾款；不得以“默认验收”替代关键成果确认。

【POLICY-FUND-005】收款账户核验  
付款收款账户名称应与合同相对方名称保持一致。存在代收、关联方收款或账户不一致的，应完成专项审批和证明留存。

【POLICY-FUND-006】高风险付款人工复核  
出现高预付款、主体不一致、自动续约、高金额框架协议、争议条款异常等情形时，应提交财务和法务联合复核。
